## 1. 产品概述

### 1.1 产品愿景

打造一款轻量、高效、适合课堂教学场景的英文单词竞赛游戏。教师通过导入单词表快速创建比赛，学生在同一台设备上同屏竞技，以"消消乐"的趣味交互形式完成单词中英文配对，激发学生学习兴趣，提升课堂参与度。

### 1.2 核心价值

- **对教师**：快速备课（导入 Excel 即可生成比赛）、课堂秩序可控（暂停/放弃等裁判操作）、直观看到学生单词掌握情况
- **对学生**：游戏化学习体验、即时反馈（消对/消错）、竞争激励（排名 + 用时展示）

### 1.3 应用场景

英语课堂中，教师邀请 1-3 名学生上讲台，在同一台设备（大屏/互动白板/电脑）上进行单词配对竞赛，台下学生观赛，增强课堂互动氛围。

### 3.2 比赛设置界面

#### 3.2.1 页面元素

##### 比赛人数设置

| 选项 | 描述 |
|------|------|
| 1 人 | 单人练习/比赛模式 |
| 2 人 | 双人同屏竞赛 |
| 3 人 | 三人同屏竞赛（最大支持人数） |

- 交互方式：分段控制器或大尺寸单选按钮，便于教师快速切换
- 默认值：2 人

##### 单词数量设置

| 字段 | 约束 |
|------|------|
| 单词数量 | 整数输入，≥ 10，≤ 导入单词表的总单词数 |
| 默认值 | 10 |

- 说明：此数值决定每位选手比赛时需要消除的单词块数量

##### 单词表导入

| 字段 | 描述 |
|------|------|
| 导入方式 | 点击上传 Excel 文件 (.xlsx / .xls) |
| 文件格式要求 | 至少两列：英文单词（列 A）、中文释义（列 B） |
| 预览区域 | 导入成功后，显示已导入单词总数、前若干行预览 |
| 校验提示 | 若文件格式不符或单词数不足设置的数量，给出明确错误提示 |

##### 设置总览

| 信息 | 内容 |
|------|------|
| 比赛人数 | N 人 |
| 每人单词数 | N 个 |
| 单词表来源 | 文件名.xlsx（共 N 个单词） |

#### 3.2.2 操作按钮

| 按钮 | 行为 |
|------|------|
| 「开始比赛」 | 校验设置完整性 → 进入比赛界面 |
| 「返回」 | 返回开始界面 |

#### 3.2.3 校验规则

1. 必须导入单词表，否则「开始比赛」按钮置灰
2. 导入的单词总数必须 ≥ 设置的单词数量
3. 单词数量必须 ≥ 10
4. 不满足条件时，在按钮旁给出提示文案

### 3.3 比赛界面

#### 3.3.1 整体布局

```
┌──────────────────────────────────────────────┐
│  [暂停]  [选手1放弃]  [选手2放弃]  ...  [返回] │  ← 裁判工具栏
├──────────────────────────────────────────────┤
│  选手1 计时: 00:12.3   │  选手2 计时: 00:18.6 │  ← 计时器行
├──────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐ │ ┌─────────┐  ┌─ │
│  │ apple   │  │ abandon │ │ │ book    │  │ s │
│  │ 苹果    │  │ 放弃    │ │ │ 书      │  │  │
│  └─────────┘  └─────────┘ │ └─────────┘  └─ │
│  ┌─────────┐  ┌─────────┐ │ ┌─────────┐  ┌─ │
│  │ cat     │  │ dog     │ │ │ chair   │  │  │
│  │ 猫      │  │ 狗      │ │ │ 椅子    │  │  │
│  └─────────┘  └─────────┘ │ └─────────┘  └─ │
│       选手1 区域          │    选手2 区域     │
└──────────────────────────────────────────────┘
```

#### 3.3.2 比赛前倒计时

- 点击「开始比赛」后，全屏显示倒计时动画：
  - **3** → **2** → **1** → **GO!**
  - 每步持续 1 秒
  - 大字居中，配合音效或视觉放大动效
- 倒计时期间，选手可以看到自己的单词块区域，但不能点击操作
- 倒计时结束后，计时器开始计时，选手可进行操作

#### 3.3.3 选手答题区域

##### 多人布局

| 人数 | 布局方式 |
|------|---------|
| 1 人 | 单区域居中，占满可用宽度 |
| 2 人 | 左右并排，垂直分割线分隔 |
| 3 人 | 左/中/右三等分，垂直分割线分隔 |

- 每位选手有独立的答题区域，区域内有明确的选手标识（如"选手 1"及颜色区分）
- 每位选手各自拥有独立计时器

##### 单词块设计

| 属性 | 说明 |
|------|------|
| 来源 | 每位选手从导入的单词表中随机抽取 N 个单词（N = 设置的单词数量），每位选手抽到的单词可能不同 |
| 展示内容 | 每个单词生成两个独立的块：**英文块**（仅显示英文单词）和**中文块**（仅显示中文释义） |
| 排列方式 | 英文块和中文块混合随机排列，形成网格布局（如 N=10，则共 20 个块，打乱排列为 4×5 或 5×4 网格） |
| 块样式 | 卡片样式，圆角矩形，醒目的文字；英文块和中文块可通过底色/边框色等做轻微视觉区分，但不可过于明显（以免降低挑战性） |
| 状态 | 正常（可点击）、选中（高亮/边框强调）、已消除（消失并留下过渡动画） |

##### 消除规则（核心交互）

| 步骤 | 操作 | 反馈 |
|------|------|------|
| 第一步 | 选手点击一个块（英文或中文均可） | 该块进入「选中」状态，高亮显示 |
| 第二步 | 选手点击另一个块 | 系统判定两个块是否为一对（英文 ↔ 对应中文释义） |
| - | **匹配成功** | 两个块同时消除（缩放 + 淡出动画，约 300ms），不可再次点击 |
| - | **匹配失败** | 两个块短暂抖动/闪烁红色提示（约 400ms），取消选中状态，恢复原状 |
| - | 点击同一类型的块（如英文+英文） | 视为匹配失败，同上反馈 |
| 特殊 | 再次点击已选中的块 | 取消选中状态 |

#### 3.3.4 计时器

| 属性 | 说明 |
|------|------|
| 精度 | 百分之一秒（00:00.00），但显示到十分之一秒即可（00:00.0） |
| 开始时机 | 3 秒倒计时结束后自动开始 |
| 停止时机 | 该选手所有单词块全部消除时停止 |
| 位置 | 每位选手区域上方，与选手标识同行 |

#### 3.3.5 裁判工具栏（页面顶部）

| 按钮 | 行为 | 说明 |
|------|------|------|
| **暂停** | 所有选手计时器暂停，答题区域不可操作，按钮文案切换为「继续」；再次点击恢复 | 课堂管理，教师需要讲解或维持秩序时使用 |
| **选手 N 放弃** | 弹出二次确认 → 确认后该选手计时停止，其答题区域置灰/标记为"已弃权"，不参与最终排名 | 每位选手一个独立的放弃按钮 |
| **返回** | 弹出确认对话框 → 确认后返回设置界面（比赛数据不保留） | 紧急退出 |

#### 3.3.6 比赛结束条件

- **全部完成**：所有未弃权的选手全部消除完自己的单词块
- **全部结束**：每位选手都已停止计时（完成或弃权）

满足以上任一条件时，比赛自动结束，跳转至排名展示。

### 3.4 比赛结束 & 排名展示

#### 3.4.1 排名规则

| 排名依据 | 说明 |
|---------|------|
| 第一优先级 | 是否完成（完成的排在弃权的前面） |
| 第二优先级 | 用时（完成者按用时升序排列；用时短者排名靠前） |
| 弃权者 | 排在所有完成者之后，按弃权时间排序（先弃权的排前面），标记为"弃权" |

#### 3.4.2 排名展示内容

| 信息 | 说明 |
|------|------|
| 排名 | 1st / 2nd / 3rd，金银铜牌图标 |
| 选手标识 | 选手 1 / 选手 2 / 选手 3，及其对应颜色 |
| 用时 | 00:12.3（完成者）或"弃权"（未完成者） |
| 特殊标注 | 冠军高亮/放大展示 |

#### 3.4.3 操作按钮

| 按钮 | 行为 |
|------|------|
| 「再来一局」 | 使用相同设置重新开始（重新随机抽词） |
| 「返回设置」 | 返回设置界面，可修改设置 |
| 「返回首页」 | 返回开始界面 |

## 4. 数据流程

### 4.1 比赛数据流

```
Excel 单词表（导入）
    │
    ▼
单词池（内存中存储，去重后的全部单词对）
    │
    ▼ 随机抽取 N 个
每位选手的单词列表（选手间独立抽取，可能不同）
    │
    ▼ 拆分为英文块 + 中文块 + 随机打乱
选手答题区的块网格
    │
    ▼ 选手操作（选中 → 匹配判断 → 消除/失败）
计时器 | 裁判操作（暂停/放弃）
    │
    ▼ 所有选手结束
排名计算 → 结果展示
```

### 4.2 数据结构

```
WordPair {
  id: string           // 唯一标识
  english: string       // 英文单词
  chinese: string       // 中文释义
}

Block {
  id: string           // 唯一标识
  wordPairId: string    // 关联的 WordPair
  type: 'english' | 'chinese'
  text: string          // 显示的文字
  status: 'normal' | 'selected' | 'matched' | 'wrong'
}

Player {
  id: number           // 选手编号 1-3
  name: string         // 选手 N
  color: string         // 标识颜色
  blocks: Block[]       // 该选手的块列表
  time: number          // 用时（毫秒）
  finished: boolean     // 是否完成
  quit: boolean         // 是否弃权
  quitTime: number|null // 弃权时间
}
```

## 5. 非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 点击响应时间 | < 100ms（触屏场景，要求即点即响） |
| 动画流畅度 | 60fps，消除动画不掉帧 |
| 页面加载 | 首屏加载 < 2s |
| Excel 解析 | 500 行以内单词表解析 < 1s |

### 5.2 兼容性

| 平台 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Edge 90+、Safari 14+ |
| 设备 | PC、教学一体机、交互式电子白板 |
| 触屏 | 完整支持触摸操作（click/touch 兼容） |
| 分辨率 | 最低 1366×768，推荐 1920×1080 |

### 5.3 可用性

- 所有可点击元素最小触控区域：44×44px（符合 WCAG 触摸目标标准）
- 裁判按钮与选手答题区之间有足够的防误触间距
- 关键操作（放弃、返回）必须有二次确认
- 选手区域之间有明显视觉分隔，避免选手误触他人区域

## 7. 异常场景处理

| 场景 | 处理方式 |
|------|---------|
| Excel 文件解析失败 | Toast 提示"文件格式不支持，请上传 .xlsx 或 .xls 文件" |
| Excel 中单词数 < 设置数量 | 提示"单词表仅有 N 个单词，请将比赛单词数设置为 N 或以下" |
| Excel 缺少必要列 | 提示"请确保 Excel 第一列为英文单词，第二列为中文释义" |
| 比赛中途误触返回 | 弹出确认对话框："比赛进行中，确定要退出吗？当前比赛数据将不会保存" |
| 选手误触放弃 | 弹出确认对话框："确定选手 N 放弃比赛吗？" |
| 浏览器刷新/关闭 | （MVP 阶段可暂不处理，后续可加 beforeunload 拦截） |

## 10. 术语表

| 术语 | 定义 |
|------|------|
| 单词块 / Block | 显示在答题区的一个可点击卡片，包含英文单词或其中文释义 |
| 单词对 / WordPair | 一个英文单词与其对应的中文释义组成的配对 |
| 消除 | 成功匹配英文块与其中文块后，两个块从答题区消失 |
| 裁判 | 操作教师，拥有暂停、放弃选手等控制权限 |
| 选手 | 参与比赛的学生 |
