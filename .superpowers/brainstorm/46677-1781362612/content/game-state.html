客户端存储策略

localStorage / IndexedDB 分工
存储引擎 存储内容 原因
localStorage 设置偏好（人数、单词数）
最近导入的单词表缓存		 小数据量，同步读取，恢复设置快
IndexedDB 单词库（多套单词表）
比赛历史记录（v1.1）		 大数据量，支持索引查询，异步不阻塞 UI
v1 预留接口，实际先走内存

v1 优先策略：设置页导完 Excel → 解析后存 setupStore (Zustand) → 同时写一份到 localStorage 做持久化缓存。
比赛数据（选手状态/计时/排名）全程在 gameStore 内存中，不落盘。
